## 🛠️ 주요 기능 구현 상세

### 1. 보안 및 인증 (Security)
*   **교사 가입 제한**: 관리자(교사) 계정 생성 시 특정 인증 코드(`2024teacher`)를 입력해야만 가입이 가능하도록 로직을 강화했습니다.
*   **자동 로그아웃**: 사용자가 15분 동안 아무런 활동이 없을 경우, 보안을 위해 자동으로 로그아웃 처리되는 아이들 타이머(Idle Timer)를 도입했습니다.
*   **실시간 버전 감시**: Firestore의 `config/app` 문서를 감시하여, 새로운 버전 배포 시 모든 사용자의 브라우저를 자동으로 새로고침합니다.

### 2. 데이터 관리 및 분석 (Data & Stats)
*   **Chart.js 대시보드**: 텍스트 위주의 대시보드를 시각화하여, 인기 비품 비중(Pie Chart)과 재고 현황(Bar Chart)을 직관적으로 파악할 수 있게 했습니다.
*   **CSV 내보내기 개선**: 엑셀에서 한글이 깨지거나 날짜가 `#####`로 나오는 문제를 해결하기 위해 BOM(Byte Order Mark), 필드 따옴표 감싸기, 상세 시간 출력 기능을 적용했습니다.

### 3. 사용자 경험 (UI/UX)
*   **Liquid Glass 테마**: 전체 시스템에 일관된 반투명 블러 효과와 세련된 그라데이션을 적용하여 프리미엄한 디자인을 유지하고 있습니다.
*   **배경화면 커스텀**: 사용자가 직접 원하는 이미지를 업로드하여 배경을 바꿀 수 있으며, 이는 브라우저에 저장되어 재접속 시에도 유지됩니다.
*   **스플래시 화면**: 앱 초기 구동 시 브랜드 이미지를 노출하며, 로딩 완료 후 부드럽게 로그인 화면으로 전환됩니다.

## 🔍 주요 문제 해결 기록 (Troubleshooting)

### Q1. "db.users is not iterable" 에러 발생
*   **원인**: Firestore 리스너가 데이터를 가져오기 전에 UI 렌더링 함수가 실행되어 데이터 형식이 맞지 않음.
*   **해결**: `db.users` 초기값을 빈 배열로 설정하고, 데이터 로드 완료 후에만 렌더링이 실행되도록 비동기 제어 로직을 수정했습니다.

### Q2. CSV 파일의 날짜가 "#####"로 표시됨
*   **원인**: 엑셀의 열 너비 문제 및 날짜 데이터 자동 인식 오류.
*   **해결**: 모든 데이터를 따옴표(`" "`)로 감싸고, 날짜 앞에 강제로 텍스트임을 인식시키기 위한 처리를 더해 해결했습니다.

### Q3. 업데이트 후 프로그램이 켜지지 않음
*   **원인**: 코드 수정 과정에서 불필요한 중괄호(`}`)가 삽입되어 자바스크립트 구문 오류(Syntax Error) 발생.
*   **해결**: 브라우저 콘솔 디버깅을 통해 해당 위치를 파악하고 즉시 제거하여 복구했습니다.
